# 개인정보 처리 위탁 계약서 유불리 검토

> 검토 대상: Proby_개인정보_처리_위탁_계약서_260303_markup (힐링페이퍼–포뮬라엑스)  
> 관점: **위탁자(고객사)** vs **수탁자(포뮬라엑스/Proby)**

## 1. 위탁자(고객사) 입장에서의 유리·불리

### 유리한 점

| 항목 | 내용 |
|------|------|
| **목적 외 이용·AI 학습 금지** (제3조) | 수탁자가 위탁 개인정보·인터뷰 결과물을 서비스 고도화·AI 학습에 쓸 수 없음. 익명·집계 통계만 허용되어 정보주체 보호에 유리함. |
| **재위탁 제한** (제4조) | 재위탁은 사전 서면 승낙 없이 불가. 클라우드·AI API 등 기술 인프라는 “재위탁 아님”으로 명시되어 수탁자 운영은 가능하되, 실질적 업무 이전은 통제 가능. |
| **인프라 제공자 변경 시 14일 사전 통보** (제4조) | 별첨 목록 외 인프라 제공자 변경 시 14일 전 서면 통보. 위탁자가 대응·검토할 시간 확보. |
| **재수탁자/인프라 제공자 책임** (제4조) | 재수탁자·기술 인프라 제공자의 고의·과실로 보안 사고 발생 시 수탁자가 위탁자에 대해 민·형사상 책임 부담. 위탁자는 직접 제3자를 상대하지 않아도 됨. |
| **관리·감독·점검 권한** (제7조) | 처리 현황, 접근 현황, 암호화 등 안전조치 이행 여부 등을 요구·점검 가능. 7영업일 전 서면 통보 후 점검 가능. |
| **교육 의무 대체** (제7조) | 수탁자가 전문 기관 점검·교육을 이수하면 위탁자 교육 수검 의무를 대체할 수 있어, 위탁자 부담 완화. |
| **해지·해제** (제9조) | 14일 시정 최고 후 미시정, 개인정보보호법령 중대 위반 등 명확한 사유로 즉시 해제·해지 가능. 본 계약 해지 시 기본 계약도 해지 가능. |
| **구상권** (제8조) | 정보주체 등 제3자에게 위탁자가 먼저 배상한 경우, 수탁자에게 구상 가능. |
| **계약 후 의무 유지** (제10조) | 계약 종료 후에도 위반에 따른 책임·비밀유지 의무는 소멸하지 않음. |

### 불리한 점

| 항목 | 내용 |
|------|------|
| **인프라 제공자 거부 제한** (제4조) | 기술 인프라 제공자(클라우드, AI API 등) 변경 시 위탁자는 “정당한 사유 없이” 거부할 수 없음. 보안·규제상 거부 사유가 애매하면 수탁자 변경을 막기 어려움. |
| **손해배상 한도** (제8조) | 수탁자 손해배상 책임 한도가 **직전 12개월 용역 대금 총액**. 대형 유출·집단 피해 시 한도가 상대적으로 낮을 수 있어, 위탁자가 추가 부담을 질 가능성 있음. |
| **고의·중과실 시 한도 배제** (제8조) | “고의 또는 중과실”이면 한도 배제되나, 입증 책임·기준이 불명확하면 분쟁 시 해석 다툼 가능. |
| **민원 창구** (제6조) | 정보주체가 수탁자에게 직접 민원(열람·정정·삭제·처리정지)을 넣을 수 있도록 수탁자가 창구를 마련해야 함. 위탁자 통제 밖에서 민원이 처리될 수 있음. |

## 2. 수탁자(포뮬라엑스/Proby) 입장에서의 유리·불리

### 유리한 점

| 항목 | 내용 |
|------|------|
| **기술 인프라 활용 명시** (제4조) | 클라우드·AI API 등 활용이 “재위탁”에 해당하지 않음이 명시되어, 사전 승낙 없이 인프라·API 교체·추가가 가능. |
| **인프라 변경 시 거부 제한** (제4조) | 위탁자가 “정당한 사유 없이” 인프라 제공자 변경을 거부할 수 없어, 운영 유연성 확보. |
| **손해배상 한도** (제8조) | **직전 12개월 용역 대금 총액**을 한도로 책임이 제한됨. 대규모 사고 시에도 한도 내 책임으로 예측 가능성 확보. |
| **익명·집계 활용 허용** (제3조) | 개인정보를 복원 불가능하게 익명처리·집계·통계화한 경우, 수탁자의 서비스 개선·연구 목적 활용 가능. |
| **교육 의무 대체** (제7조) | 전문 기관 점검·교육 이수로 위탁자 요구 교육 수검 의무를 대체할 수 있어, 운영 부담 완화. |
| **3개월 자동 파기 원칙** (제6조) | 별도 통지가 없으면 수집일로부터 3개월 경과 시 파기하는 것이 원칙이라, 보관 기간·범위가 명확함. |
| **관할 법원** (제12조) | 서울중앙지방법원 관할로 합의되어, 수탁자 소재지·업무와 관련해 소송 시 불리하지 않을 수 있음. |

### 불리한 점

| 항목 | 내용 |
|------|------|
| **재수탁자/인프라 제공자 책임 전가** (제4조) | 재수탁자·기술 인프라 제공자의 고의·과실로 인한 사고에 대해 수탁자가 위탁자에 대해 전부 책임. AWS·AI API 제공자 등 제3자 사고도 수탁자가 최종 부담. |
| **동등 수준 계약 의무** (제4조) | 재수탁자·기술 인프라 제공자와 “본 계약과 동등한 수준”의 개인정보 보호 조치·관리 의무를 계약에 명시해야 함. 제3자 계약 관리 부담 증가. |
| **연 1회 이상 점검** (제4조) | 재수탁자·인프라 제공자에 대해 연 1회 이상 점검 실시, 위탁자 요구 시 점검 결과 보고서 제출 의무. |
| **광범위한 안전성 확보조치** (제5조) | 개인정보보호법령상 기술적·관리적 조치를 모두 이행해야 하며, 법령 개정 시 그에 맞춰 조치 범위가 확대될 수 있음. |
| **7영업일 전 통보 후 점검** (제7조) | 위탁자가 실태 점검을 요구할 때 7영업일 전 서면 통보만 하면 되므로, 수탁자는 점검에 응해야 함. |
| **해지 시 기본 계약 연동** (제9조) | 본 계약 해지 시 위탁자가 기본 계약도 해지할 수 있어, 개인정보 위탁 계약이 전체 거래 관계에 영향. |
| **비밀유지·책임의 계속** (제10·11조) | 계약 종료 후에도 위반에 따른 책임·비밀유지 의무가 남아 지속적 준수 필요. |

## 3. 공통·보완 관점

### 계약서 자체의 장점

- 개인정보보호법·정보통신망법 등 법령과 정합성이 높고, 위탁·재위탁·파기·안전조치 등 핵심 요소가 빠짐없이 반영되어 있음.
- “기술 인프라 활용 = 재위탁 아님”이 명시되어, AI·클라우드 서비스 구조와 현실에 맞게 작성됨.
- 익명·집계 통계만 수탁자 활용 허용하는 부분이 명확해, 정보주체 보호와 수탁자 활용 범위의 균형이 잘 잡혀 있음.

### 검토·보완 권고 사항

| 구분 | 권고 |
|------|------|
| **위탁자** | (1) 손해배상 한도(12개월 용역 대금)가 사업 규모·리스크에 비해 적정한지 검토. (2) 기술 인프라 제공자 “정당한 사유”의 구체적 예시(보안·규제·지역 제한 등)를 부록이나 합의서에 적어 두는 것 검토. (3) 민원 창구는 수탁자 마련이지만, 위탁자에게 통보·공유 방식 합의 권장. |
| **수탁자** | (1) 재수탁자·클라우드/AI API 제공자와의 계약에 “위탁자와 체결한 위탁계약과 동등한 수준” 조항·실무 체크리스트를 반영. (2) 연 1회 점검·보고서 양식·제출 시한을 사전에 정해 두면 이행 부담이 줄어듦. (3) 한도 배제되는 “고의·중과실”의 판단 기준을 내부 가이드로 정리해 두면 분쟁 시 유리. |
| **공통** | (1) 제4조 “별첨 목록”(기술 인프라 제공자 목록)을 계약 체결 시 반드시 작성·첨부. (2) 제6조 보유 기간 연장 “서면 통지”의 방식(이메일·문서함 등)을 사전에 합의해 두면 분쟁 예방에 도움. |

## 4. 요약

| 관점 | 종합 |
|------|------|
| **위탁자** | 목적 외 이용·AI 학습 금지, 재위탁 통제, 관리·감독·점검·구상권·해지권 등으로 **전반적으로 유리**한 구조. 다만 인프라 변경 거부가 제한되고, 손해배상 한도가 상대적으로 낮을 수 있어 **한도·인프라 변경 사유**는 추가 검토 권장. |
| **수탁자** | 기술 인프라 활용·손해배상 한도·익명·집계 활용·교육 대체 등으로 **운영상 유리**한 조항이 많음. 대신 재수탁자/인프라 제공자에 대한 **전면 책임·동등 수준 계약·점검 의무** 부담이 커서, 제3자 계약·점검 체계를 잘 갖춰 두는 것이 중요. |
| **계약 균형** | 법령 반영과 조항 구성이 잘 되어 있어, 위탁자·수탁자 모두 수용 가능한 수준. 위탁자는 “한도·인프라 거부 사유”, 수탁자는 “제3자 계약·점검·고의·중과실 기준”을 보완하면 리스크 관리에 도움이 됨. |
