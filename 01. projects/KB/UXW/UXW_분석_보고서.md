# KB 뱅킹 앱 홈 화면 UXW 평가 분석 보고서

**작성일:** 2026년 3월 22일  
**조사 유형:** 1:1 심층 인터뷰 (AI 모더레이터)  
**참여자 수:** 2명  
**분석 관점:** UX Writing (문구 이해도 · CTA 클릭 의도 · 버전 선호도)

---

## 1. 조사 개요

### 조사 목적
KB 뱅킹 앱 홈 화면의 자산/지출 영역에 적용된 **기존안**과 **개선안**의 UXW(문구, CTA 버튼 라벨)에 대한 유저 반응을 비교 평가한다. 구체적으로 ①문구 이해도, ②기능 기대치 형성, ③클릭 의도 유발 여부를 측정한다.

### 비교 대상

| 영역 | 기존안 | 개선안 |
|------|--------|--------|
| **자산 영역 설명 문구** | "총 자산 / 내가 모은 자산은 얼마일까요?" | "예적금, 각종 포인트 등 잊고 있던 자산 없을까?" |
| **자산 영역 CTA** | 자산 확인하기 | 내 숨은 자산 찾기 |
| **지출 영역 설명 문구** | "지출 정보 / 수입과 지출을 한눈에 My 현금흐름으로 시작하세요!" | "이번 달 수입·지출, 카드 대금 한눈에 보려면?" |
| **지출 영역 CTA** | 자산 연결하기 | 내 현금 흐름 보기 |

## 3. 주요 발견사항

### 3-1. 화면 전반 첫인상

**공통 반응:**
- 두 참여자 모두 화면을 KB 뱅킹 앱으로 인식
- 기존에 쓰는 앱(토스뱅크) 대비 정보 구성이 다르다고 느낌
- 버튼이 "버튼처럼 보이지 않는다"는 지적이 두 참여자 모두에게서 반복됨

> **참석자 #2:** "자산 확인하기 버튼이 좀 버튼처럼 보이지 않아서 클릭할 것 같은 느낌이 좀 안 들고요."  
> **참석자 #1:** "정보가 많이 간추려졌고, 제가 사용하던 메인 홈페이지는 아닌 것 같습니다."

**UXW 시사점:** CTA 버튼의 시각적 표현(affordance)이 클릭 의도에 직접 영향을 줌. 문구보다 버튼 스타일이 먼저 필터링됨.

---

### 3-2. 자산 영역 문구 비교

#### 문구 이해도

| 참여자 | 기존안 ("자산 확인하기") | 개선안 ("내 숨은 자산 찾기") |
|--------|--------------------------|------------------------------|
| 참석자 #1 | 모든 은행 자산을 연동 후 통합 조회하는 기능으로 이해 | 포인트 소멸·오래된 계좌를 찾아주는 기능으로 이해 |
| 참석자 #2 | 예금·적금·퇴직연금 등 자산별 분류 조회로 이해 | 단순 예금·적금 외 각종 포인트까지 포함한 광범위한 자산 통합 조회로 이해 |

**핵심 차이:** 기존안 "자산 확인하기"는 기능 자체는 명확하지만 **'이미 알고 있는 것'을 재확인하는 느낌**으로 클릭 동기가 약함. 개선안 "숨은 자산 찾기"는 **잊고 있었거나 소멸 위기에 처한 자산**이라는 구체적 연상을 유발하여 더 강한 흥미를 만들어냄.

#### CTA 클릭 의도

| 참여자 | 기존안 "자산 확인하기" | 개선안 "내 숨은 자산 찾기" |
|--------|------------------------|---------------------------|
| 참석자 #1 | 상단 노출되어 있으니 눌러야 할 것 같음 (수동적 동기) | **더 누르고 싶다** — "포인트 소멸 기간이 남아있지 않는 것들도 있을 것 같아서 찾아주면 굉장히 도움이 될 것 같다" |
| 참석자 #2 | 이미 토스에서 관리 중이라 굳이 안 해도 됨. "팝업이 많이 나올 것 같다"는 우려. 기존안 "자산 확인하기"를 개인 CTA 선호 항목으로 선택 | 반반 — "숨은 자산이 별로 없을 것 같다. 이벤트가 있다면 할 수도 있다" |

> **참석자 #1(개선안 반응):** "제가 포인트 소멸 기간이 남아있지 않는 것들도 있을 것 같은데 그거를 찾아주면 굉장히 도움이 될 것 같습니다."

**인사이트:** 개선안 "내 숨은 자산 찾기"는 **실질적 혜택 기대(잃어버린 돈 찾기)**를 유발하여 클릭 동기를 높임. 다만 참석자 #2처럼 이미 자산을 통합 관리하는 유저에게는 차별적 동기 부여가 약함.

### 3-3. 지출 영역 문구 비교

#### 문구 이해도

| 참여자 | 기존안 ("자산 연결하기") | 개선안 ("내 현금 흐름 보기") |
|--------|--------------------------|------------------------------|
| 참석자 #1 | 수입·지출 연동 기능 | 이번 달 수입·지출 전체 파악 기능 |
| 참석자 #2 | 지출 내역 리스트 형태로 나올 것 같은 느낌 | **대시보드 형태**로 수입/지출 추이를 시각화한 화면 기대 |

**핵심 차이:** 기존안 "자산 연결하기"는 **연결하는 과정(설정/동의)**에 초점이 맞춰져 사용자가 얻을 결과물이 아닌 프로세스 단계처럼 느껴짐. 개선안 "현금 흐름 보기"는 **추이/변화/흐름**의 시각적 이미지(대시보드)를 연상시켜 결과 가치가 명확함.

#### CTA 클릭 의도

| 참여자 | 기존안 "자산 연결하기" | 개선안 "내 현금 흐름 보기" |
|--------|------------------------|---------------------------|
| 참석자 #1 | **누르고 싶지 않음** — 이미 토스뱅크의 캘린더 형식 선호 | **누르고 싶지 않음** — 동일 이유 (두 CTA 모두 필요성 못 느낌) |
| 참석자 #2 | 이미 토스를 쓰기 때문에 필요성 낮음 | **더 흥미롭고 보고 싶다** — "흐름이라는 단어 때문에 대시보드를 기대하게 됨" |

> **참석자 #2:** "지출 연동의 경우 뭔가 내가 지출을 했던 포인트별로만 보여질 것 같은 이게 리스트처럼만 구성이 될 것 같은 느낌이었는데 내 현금 흐름을 본다라고 했을 때는 대시보드 같은 게 보여질 것 같아서 내가 실제로 얼마나 돈을 사용해 가고 있는지 그 추이를 볼 수 있겠다는 기대가 좀 들어서 더 흥미롭다고 느껴지는 것 같아요."

**인사이트:** 개선안 "내 현금 흐름 보기"는 결과 가치(무엇을 보게 되는지)를 명확히 전달. 기존안 "자산 연결하기"는 동작(연결)에 집중되어 사용자가 얻을 **결과 가치를 상상하기 어려움**.

### 3-4. 버전 전반 선호도

| 참여자 | 최종 선택 버전      | 선택 이유                                                        |
| --- | ------------- | ------------------------------------------------------------ |
| 참석자 #1 | **개선안 (왼쪽)**  | "더 자세하게 표현되어 있고, 숨은 자산이 궁금하고 가계부 역할을 할 것 같다"                 |
| 참석자 #2 | **기존안 (오른쪽)** | "총 자산, 지출 정보라는 레이블이 명확하고, 자산 확인하기·자산 연결하기 버튼 문구가 이야기가 쉬운 느낌" |

**참고:** 참석자 #2은 기존안 전체를 선택했지만, 지출 영역 개별 CTA로는 개선안의 "내 현금 흐름 보기"를 선호함. 즉 **섹션 레이블·전체 구조(기존안 우위)와 지출 CTA 문구(개선안 우위)에 대한 평가가 분리**됨.

## 4. UXW 관점 종합 인사이트

### 인사이트 1. 기존안 레이블은 명확하지만, 개선안 문구가 클릭 의도를 더 강하게 유발한다

기존안의 "총 자산", "지출 정보" 레이블과 "내가 모은 자산은 얼마일까요?" 문구는 정보 구조를 명확히 전달하나, 개선안의 "예적금, 각종 포인트 등 잊고 있던 자산 없을까?"는 **사용자의 pain point(잊고 있던 돈)를 직접 자극**하여 행동 동기를 높인다. UXW에서 **혜택 중심·질문형 카피**는 단순 설명형보다 행동 유도력이 높다.

### 인사이트 2. CTA 라벨은 '결과 가치'를 전달해야 한다

- 기존안 "자산 연결하기" → 사용자 입장에서 **해야 하는 일(과정)** 처럼 느껴짐  
- 개선안 "내 현금 흐름 보기" → 누르면 **무엇을 얻는지(결과)** 가 명확함  
- 개선안 "내 숨은 자산 찾기" → **기대 효용(놓친 것을 찾아줌)** 이 내포됨  
- 기존안 "자산 확인하기" → 범용적이고 무난하지만 차별화된 행동 동기 약함  

결과 지향 CTA(예: "보기", "찾기")가 과정 지향 CTA(예: "연결하기", "확인하기")보다 클릭 의도를 더 강하게 유발함.

### 인사이트 3. 섹션 레이블과 CTA 문구의 역할은 다르다

기존안의 "총 자산", "지출 정보" 레이블은 **정보 구조 파악**에 도움을 주며, 참석자 #2이 기존안 전체를 선택한 이유가 됨. 반면 개선안의 CTA("내 숨은 자산 찾기", "내 현금 흐름 보기")는 **행동 유도** 측면에서 우위. 이상적인 UXW 설계는 **명확한 레이블(기존안) + 혜택 중심 CTA(개선안)**를 결합하는 방향.

### 인사이트 4. 개인정보 동의 절차에 대한 인지적 저항이 클릭 억제 요인

두 참여자 모두 자산 연동 기능에 대해 "개인정보 동의 팝업이 많이 나올 것 같다"는 우려를 공통적으로 언급. 문구 자체의 매력도가 높아도 **프로세스 복잡성에 대한 선입견**이 클릭 의도를 낮춤. 온보딩 단계의 UXW(예: "30초면 연결 완료") 보완이 병행되어야 함.

### 인사이트 5. 이미 유사 서비스 이용자는 중복 필요성을 느끼지 않는다

두 참여자 모두 토스뱅크에서 자산 통합 관리를 하고 있었으며, KB 앱의 자산 연동 기능에 대한 클릭 동기가 낮음. 기능 UXW 외에도 **KB만의 차별점**을 문구로 전달하는 전략이 필요함.

## 5. 제언

### 단기 개선안 (문구 조정)

| 영역 | 기존안 | 개선안 | 제안 방향 |
|------|--------|--------|-----------|
| 자산 CTA | 자산 확인하기 | **내 숨은 자산 찾기** | 개선안 유지 — 혜택 기대 유발 효과가 명확히 우위 |
| 지출 CTA | 자산 연결하기 | **내 현금 흐름 보기** | 개선안 유지 — 결과 가치 전달 측면에서 우위 |
| 자산 섹션 레이블 | 총 자산 | 예적금·포인트 잊고 있던 자산... | **기존안 유지** (명확성·구조 이해 우위) |
| 지출 섹션 레이블 | 지출 정보 | 이번 달 수입·지출... | **기존안 유지** (명확성·구조 이해 우위) |

### 구조적 UXW 제안

1. **레이블(기존안) + 혜택형 CTA(개선안) 혼합 구조** 채택  
   예: `총 자산 / 내가 모은 자산은 얼마일까요?` + CTA: **내 숨은 자산 찾기**  
   예: `지출 정보 / 수입과 지출을 한눈에 보려면?` + CTA: **내 현금 흐름 보기**

2. **진입 장벽 낮추는 마이크로카피 추가**  
   "자산 확인하기" 버튼 하단에 "3초 만에 연결 가능" 등 프로세스 간소화 암시 문구 추가 검토

3. **KB 고유 차별점 문구화**  
   이미 타 앱에서 자산 관리 중인 유저에게 "KB에서만 확인 가능한 정보"를 문구로 어필하는 방향 검토
