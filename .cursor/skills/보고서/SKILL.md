# /보고서 스킬

인터뷰 녹취록·리서치 MD 파일을 읽어 MECE 구조의 인터랙티브 HTML 보고서를 생성한다.
레퍼런스 템플릿: `ongoing/LG Uplus/Poc2/녹취록_정리/POC2_녹취록_종합요약_보고서.html`

## 트리거

사용자가 `/보고서 {파일 또는 폴더 경로}` 형태로 입력한다.
- 예: `/보고서 ongoing/LG Uplus/Poc2/녹취록_정리/`
- 예: `/보고서 60. 유저인사이트팀/스푼랩스/인터뷰_정리.md`
- 파일명만 주어지면 워크스페이스에서 경로를 검색해 확정한다.

## 동작 순서

### 1단계: 소스 파일 확정
- 폴더 경로이면 → 폴더 내 모든 `*_녹취록.md` 또는 `*.md` 파일을 수집한다.
- 단일 파일이면 → 해당 파일 1개를 분석한다.
- 브랜드 이미지(`.jpg`, `.png`)가 같은 폴더에 있으면 로고로 사용한다.

### 2단계: 콘텐츠 파싱

**인터뷰 녹취록 MD (HTML table 형식)**
- `<th>` 셀에서 화자 판별: `AI` → AI 인터뷰어, `인터뷰어` → 인간 인터뷰어, `참여자` → 참여자
- `## 추가 인터뷰` 섹션이 있으면 해당 참여자를 **추가인터뷰 대상**으로 마킹
- 참여자 발화에서 키워드별 인용구를 추출한다

**일반 리서치 MD**
- `##` 섹션 헤더 → MECE 프레임 주제로 매핑
- 인용구는 `>` blockquote 또는 따옴표 안 발화를 추출한다

### 3단계: MECE 분석 프레임 설계

데이터 성격에 따라 MECE 섹션을 설계한다.

| 데이터 유형 | 권장 MECE 구조 |
|---|---|
| 인터뷰 녹취록 | 채널선택 → 요금제/조건 → 결합서비스 → 부가서비스 → 멤버십/혜택 → 온보딩 → 네트워크 → 장기관계 |
| UX 리서치 | 태스크 인식 → 탐색 패턴 → 정보 구조 → 인터랙션 → 감성 반응 |
| 브랜드 조사 | 인지도 → 연상 이미지 → 경험 평가 → 경쟁 비교 → 추천 의향 |
| 기타 리서치 | 파일 내 섹션 구조를 최대한 반영해 4–8개 프레임으로 압축 |

### 4단계: HTML 보고서 생성

레퍼런스 HTML과 **동일한 시각 구조**를 유지한다.

**필수 구성 요소:**
- 헤더: 프로젝트명, 참여자 수, 수집일, 로고(있으면)
- Stats bar: 참여자 수, 추가 인터뷰 수, 프레임 수, 인용구 수
- 참여자 칩 그리드 (추가 인터뷰는 주황색 강조)
- 좌측 스티키 사이드바 네비게이션
- **[최상단 필수] 핵심 개선 과제 섹션** (아래 상세 참조)
- 각 MECE 섹션: insight-box(핵심 발견) + 인용구 카드(3–5개) + 표/그래프(선택)
- **추가 인터뷰 콜아웃**: `★ 추가 인터뷰` 주황색 박스로 심층 발견사항 강조
- 페인포인트 빈도 히트맵 (가로 막대)
- 개선 제언 카드 (즉시 적용 / 중장기)

**핵심 개선 과제 섹션 (MECE 섹션보다 먼저, 본문 최상단에 위치):**

레이아웃: **하나의 통합 컨테이너** 안에 5개 행을 쌓는 테이블형 구조. 개별 카드를 따로 나열하지 않는다.

```
┌─────────────────────────────────────────────────────────────┐
│  [헤더] {브랜드/프로젝트명} 핵심 개선 과제 5                       │
├──────────────────────┬──────────────────────────────────────┤
│ 좌측 (220px 고정)      │ 우측 (flex:1)                        │
│ - 번호 배지 + 카테고리  │ - 인용구 카드 2개                      │
│ - 과제명 (굵게)        │   (border-left 컬러 바 + 이탤릭)       │
│ - 한 줄 설명           │                                      │
│ - 📈 예상 임팩트 박스   │                                      │
├──────────────────────┴──────────────────────────────────────┤ ← 구분선
│ (다음 행 반복 × 5)                                             │
└─────────────────────────────────────────────────────────────┘
```

- 컨테이너: `border: 1.5px solid var(--border); border-radius:16px; overflow:hidden;`
- 각 행: `display:grid; grid-template-columns:220px 1fr; border-bottom:1px solid var(--border);`
- 마지막 행(5번)은 `border-bottom` 없음
- 좌측 배경: 과제별 파스텔 컬러 (`linear-gradient`) + `border-right:1px solid var(--border)`
- 번호 배지: `width:28px; height:28px; border-radius:8px;` — 과제별 다른 색
- 예상 임팩트 박스: `background:#e8f5e9; border-radius:8px;` (초록, 모든 행 동일)
- 인용구 카드: `background`는 해당 과제 색조의 연한 버전 + `border-left:3px solid 해당색`
- 사이드바 최상단에 `★ 핵심 개선 과제 5` 링크 추가 (primary 컬러 강조 스타일)

**인용구 규칙:**
- 섹션당 3–5개
- 각 참여자에서 최소 2개 이상 사용
- 핵심 단어는 `<strong>` 태그로 강조

**브랜딩:**
- 로고 이미지가 있으면 사용, 없으면 프로젝트명 텍스트로 대체
- Primary 컬러는 기본 `#E6007E` (LG U+ 마젠타). 사용자가 다른 브랜드 컬러를 지정하면 해당 컬러 사용
- 추가 인터뷰 콜아웃 색상: `#ff8f00` (주황) 고정

### 5단계: 파일 저장 및 안내

- 저장 위치: 소스 파일과 **같은 폴더**에 `{프로젝트명}_종합요약_보고서.html`
- 사용자가 `저장:` 또는 `파일명:` 으로 지정하면 그에 따른다
- 생성된 파일 경로와 섹션 수·인용구 수를 간단히 안내한다

## 옵션

사용자가 같은 메시지에서 아래를 지정하면 그에 따른다:

| 옵션 | 예시 |
|---|---|
| 컬러 지정 | `컬러: #0066cc` |
| 파일명 지정 | `파일명: 스푼랩스_보고서.html` |
| 로고 지정 | `로고: assets/logo.png` |
| 프레임 직접 지정 | `프레임: 가입동기, 불만, 개선점` |

## 참고

- 추가 인터뷰 여부 판단: `인터뷰어` 화자 발화 수 > 5개이면 추가인터뷰 대상으로 간주
- 인용구 품질 기준: 30자 이상, 구체적 경험·감정·수치가 포함된 발화 우선 선택
- 페인포인트 빈도는 섹션별 분석 후 상위 10개를 히트맵으로 시각화
